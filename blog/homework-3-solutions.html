Question 1¶

(a) Which part of the integrand is more suitable as the function to evaluate, and which is best used as a distribution to draw random variates from? Explain why.

We split the integrand as $f(x) = h(x)g(x)$. In choosing which part of the integrand is more suitable to evaluate and which as a distribution from which to draw, we note that the distribution should be easy to find a normalization factor for (not necessary in general, but in this case this is possible and this way we don’t have to approximate the normalization say with Monte Carlo since we can just get it analytically) and the new integrand should be ''flatter'' than the original one. After some experimentation, we find an acceptable candidate is $g(x) = 1/[3+(x−3)^2]$. We can invert this analytically to sample the $x_i$ from the distribution g$*$, where $ g* = Cg $ for a normalization constant C. Integrating from 0 to $ \infty $ we find that the normalization constant is $C = 6\sqrt{3}/5\pi$. Now we ￼can sample directly from g$*$ since integrating results in an easily invertible function (CDF).

(b) Implement the Metropolis-Hastings algorithm that has as a stationary distribution to sample from, $g(x)$. Run the simulation 50 times for sample size of 150,000 and report the value of the integral I and the VAR(I).

Now, we use importance sampling to evaluate the integral. Looking at a plot of the function, we see that we can use a distribution proportional to $x e^{−x}$, namely a $\Gamma(2,1)$ pdf, which has a normalizing constant of 1. So, drawing $x_i$ from a $\Gamma(2, 1)$, we have the importance sampling estimate:

$\int_{0}^{\infty} \frac{x {\rm ln}(x+1)}{3+(x-3)^2}e^{\sqrt{x}-x}dx = \frac{1}{N} \sum\limits_{i=1}^{N} \frac{{\rm ln}(x_i+1)}{3+(x_i-3)^2}e^{\sqrt{x_i}} $

We have an $m \times n$ matrix representing the pdf values in a 2-dimensional discrete space (saved as the file ‘pdf data.txt’). We implement a random walk Metropolis-Hastings algorithm to draw pairs from this distribution. We use component-wise updating. Since we draw from a uniform proposal, we use the Metropolis algorithm (and we manually restrict the ends so that we do not go off the grid, which introduces a slight error but does not prevent us from clearly seeing the message.

In [27]: 
import scipy as sp
import scipy.misc as spmisc

STEPS = 250000 # How many steps should we take
BURN_IN = 10000 # How many should we throw away for burn-in?

M,N = pdf.shape[0],pdf.shape[1] # How large is the array?

def mh_walk(step_sz):
    def proposal_dist(x,y):
        x_prop = x+sp.random.random_integers(-step_sz,step_sz)
        y_prop = y+sp.random.random_integers(-step_sz,step_sz)
        x_prop = sp.maximum(0,sp.minimum(x_prop,M-1))
        y_prop = sp.maximum(0,sp.minimum(y_prop,N-1))
        return (x_prop,y_prop)

    # Find starting value.
    x_values = sp.zeros(STEPS+1)
    y_values = sp.zeros(STEPS+1)

    x_values[0] = sp.random.random_integers(0,M)
    y_values[0] = sp.random.random_integers(0,N)

    for i in xrange(STEPS):
        x = x_values[i]
        y = y_values[i]
        x_prop,y_prop = proposal_dist(x,y)
        p_prop = pdf[x_prop][y_prop]
        p_curr  = pdf[x][y]

        a_p = (p_prop/p_curr) if (p_curr > 0) else sp.inf
        if sp.random.random() < a_p:
            x = x_prop
            y = y_prop
        x_values[i+1] = x
        y_values[i+1] = y

    # Remove burn-in.
    x_values = x_values[BURN_IN:]
    y_values = y_values[BURN_IN:]

    return (x_values,y_values)

def MH_multivariate_P3d(p,q,q2,n,gamma):
    # start from a random position 
    np.random.seed(seed=10)
    x0=0
    y0=0
    x_prev =x0
    y_prev =y0
    
    x = np.zeros(n)
    y = np.zeros(n)
    i=0
    accepted = 0
    
    while i<n:
        x_star =  q(x_prev) # specific x-direction proposal
        P_star = p(x_star,y_prev)
        P_prev = p(x_prev, y_prev)
        A =  P_star/P_prev 
        U = np.random.uniform();
        
        if (U < A):
            x[i] = x_star
            i = i + 1
            x_prev = x_star
            accepted+=1
        else: 
            x[i] = x_prev
            x_prev = x[i]
            i = i + 1
        
        i=i-1
        # now lets do the y step

        P_prev = p(x_prev, y_prev)

            i = i + 1
            y_prev = y_star
            accepted+=1
        else: 
            y[i] = y_prev
            y_prev = y[i];  
            i = i + 1
            
    acceptance_rate = accepted/float(n*2)

Question 4¶

Suppose we ask individuals to remember the chronological order of world cup winners. Individuals make a number of mistakes in the ordering of winners that can be captured by simple probabilistic models, such as Mallows model. Here's a simple example to explain Mallows model:

Suppose that we are looking at the last four winners: France, Brazil, Italy, Spain. We will represent this true ordering by a vector $\omega =$ (1,2,3,4) = (France, Brazil, Italy, Spain). Mallows model now proposes that the remembered orderings tend to be similar to the true ordering, with very similar orderings being more likely than dissimilar orderings. Specifically, according to Mallows model, the probability that an individual remembers an ordering $\theta$ is proportional to:

$ p(\theta | \omega, \lambda) \propto e^{-\lambda d(\theta, \omega)} $

In this equation, $d(\theta, \omega)$ is the Kendall tau, $\tau$ is the distance between two orderings. This distance measures the number of adjacent pairwise swaps that are needed to bring the two orderings into alignment.

For example, if $\theta = $ (Brazil, France, Italy, Spain) then $d(\theta, \omega) = 1$ because one swap is needed to make the two orderings identical. Similarly, if $\theta=$ (Brazil, Italy, France, Spain), then $d(\theta, \omega) = 2$ because two swaps are needed. Note that in Kendall tau distance, only adjacent items can be swapped. The scaling parameter $\lambda$ controls how sharply peaked the distribution of remembered orderings is around the true ordering. Therefore, by increasing $\lambda$, the model makes it more likely that the correct ordering (or something similar) will be produced. The problem is now to generate orderings $\theta$ according to Mallows model, given the true ordering $\omega$ and scaling parameter $\lambda$. This can be achieved in very simple ways using a Metropolis sampler.

Implement a Metropolis-Hastings algorithm to produce sample guesses from 100 individuals. Choose various proposal distributions and experiment with different values of $\lambda$.

from __future__ import division
import scipy
import itertools

BURN_IN   = 10**4
NUM_INDI  = 10**2 # number of individuals
NUM_ITERS = BURN_IN+NUM_INDI
LAMBDA    = 0.5
NUM_YEARS = 4

def kendall_tau2(theta):
    """We assume that omega is always in sorted order.
    This is true up to relabeling."""
    T = len(theta)
    return int(sum([1 for i in xrange(T) for j in xrange(i+1,T) 
        if theta[i] > theta[j]]))

def next_perm_swaps(sigma,num_swaps):
    """Uses a uniform distribution over permutations that differ from
    the input by num_swaps adjacent transpositions."""
    A_value = 1
    positions = scipy.random.randint(0,NUM_YEARS-1,num_swaps)
    # mock_sigma keeps track of the effect of the proposed transpositions.
    mock_sigma = list(sigma) # make a copy
    for pos in positions:
        A_value *= (scipy.exp(LAMBDA) if (mock_sigma[pos] > mock_sigma[pos+1]) 
                    else scipy.exp(-LAMBDA))
        mock_sigma[pos],mock_sigma[pos+1] = mock_sigma[pos+1],mock_sigma[pos]
    if (scipy.random.uniform() < A_value):
        sigma = mock_sigma
    return sigma

def next_perm_unif(sigma,num_swaps):
    """Uses a uniform distribution over all permutations."""
    # num_swaps is not used.
    proposed = scipy.random.permutation(NUM_YEARS)
    unscaled_pdf = lambda pi: scipy.exp(-LAMBDA*kendall_tau2(pi))
    A_value = unscaled_pdf(proposed)/unscaled_pdf(sigma)
    if (scipy.random.uniform() < A_value):
        sigma = proposed
    return sigma

def gen_perms(proposal_method,num_swaps):
    """Generates orderings using the given proposal distribution."""
    pi = scipy.random.permutation(NUM_YEARS)
    history = [[0] for i in xrange(NUM_ITERS)]
    for i in xrange(NUM_ITERS):
        A_values = [scipy.exp(LAMBDA) if (pi[j] > pi[j+1]) 
                    else scipy.exp(-LAMBDA) for j in xrange(NUM_YEARS-1)]
        pi = proposal_method(pi,num_swaps)
        history[i] = tuple(pi)
    history = tuple(history[BURN_IN:])
    print('\nTime spent in each ordering:')
    totals    = dict()
    num_perms = dict()
    for sigma in itertools.permutations(range(NUM_YEARS)):
        percent = history.count(sigma)/NUM_INDI
        dist    = kendall_tau2(sigma)
        if dist not in totals.keys():
            totals[dist] = 0
        if dist not in num_perms.keys():
            num_perms[dist] = 0
        totals[dist]    += percent
        num_perms[dist] += 1
        print(str(sigma)+': '+str(100*percent)+'%')
    print('\nKendall tau distance and estimate of scaling parameter:')
    for key in totals:
        mean = totals[key]/num_perms[key]
        param = mean/scipy.exp(-LAMBDA*key)
        print(str(key)+' adjacent transpositions: '+str(param))
    print('')
    return history
